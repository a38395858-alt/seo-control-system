# SEO 中控系统一期：关键词挖掘模块开发文档

> 版本：v1.1  
> 更新：2026-07-16  
> 范围：仅包含关键词发现、导入、清洗、管理和导出。  
> 数据源：Google Ads API、Google Ads 下载文件、Google 下拉建议词协议。

## 1. 目标与范围

本期目标是把种子词、Google Ads 数据和 Google 下拉建议词，沉淀为一个可筛选、可追溯、可复用的项目关键词库。

```text
创建挖词任务 / 上传文件
          ↓
获取原始关键词与可用指标
          ↓
清洗、过滤、去重、保存来源
          ↓
评分、人工筛选、标签、导出
```

本期不做：内容选题或生成、CMS 发布、GSC 接入、排名/流量追踪、页面审计、竞品分析、内链/锚文本、Google Ads 投放与出价操作。后续功能只读取本期沉淀的关键词库。

## 2. 数据源与数据口径

| 来源 | 接入方式 | 用途 | 可获得字段 |
| --- | --- | --- | --- |
| Google Ads Keyword Planner | 官方 Google Ads API | 主数据源 | 建议关键词、月均搜索量、月度趋势、广告竞争度、页首出价范围。 |
| Google Ads 下载文件 | CSV/XLSX 上传 | 无 API 时的降级与补充 | 用户下载的关键词规划数据。 |
| Google 下拉建议词 | 非官方协议适配器 | 长尾词发现 | 建议词、种子词、扩展规则、发现路径与时间。 |

规则：

1. Google Ads 的搜索量、竞争度、出价都是广告规划估算，不等于 SEO 排名或精确自然搜索量。
2. Google 下拉词不具备搜索量、CPC、竞争度，前端不可将空值显示为 0 或虚构数据。
3. 同一关键词在不同国家/地区、语言或日期的指标必须保存为独立快照。
4. 每条关键词必须可追溯到来源、任务/批次、种子词、市场、采集时间和原始记录。

## 3. 用户流程

### 3.1 Ads API 挖词

`选择项目 → 选择已授权 Ads 账号 → 填写种子词或 URL → 选国家/语言 → 提交任务 → 查看结果 → 确认入库`

### 3.2 Google 下拉词挖词

`选择项目 → 填写种子词 → 设国家/语言、扩展规则、层级与请求上限 → 提交任务 → 查看扩展链路 → 确认入库`

### 3.3 导入 Ads 下载文件

`选择项目 → 上传 CSV/XLSX → 预览并映射列 → 设置国家/语言/指标日期 → 校验 → 确认导入 → 下载错误行或进入词库`

## 4. 功能设计

### 4.1 项目与市场配置（P0）

项目字段：项目名称、站点名称/URL（仅标识）、默认国家/地区、默认语言、默认币种、时区、品牌词、否定词。

关键词唯一键：`project_id + normalized_keyword + country_code + language_code`。同一词的不同市场应保留为独立词条。

### 4.2 Google Ads API 连接与挖词（P0）

#### 授权与账号选择

- 使用 Google OAuth 2.0，scope：`https://www.googleapis.com/auth/adwords`。
- 平台需配置 OAuth Client、回调地址、Google Ads API Developer Token。
- 授权后只显示用户可访问的 Customer ID；MCC 场景可选择下属客户账号。
- refresh token 加密保存，前端、日志和错误信息不得暴露凭据。
- 连接状态：`未连接`、`有效`、`授权失效`、`权限不足`、`已解除`。

Google Ads 测试账号不提供真实市场数据。若 Developer Token 或用户 Ads 权限未就绪，系统仍应可使用文件导入及下拉词数据源。

#### 任务参数

调用官方 `KeywordPlanIdeaService.GenerateKeywordIdeas`。种子为互斥模式，一次任务只允许选择一种：

| 模式 | 输入 | 说明 |
| --- | --- | --- |
| 种子词 | 1–20 个词 | 围绕主题扩词。 |
| URL | 一个页面或站点 URL | 围绕现有页面扩词。 |
| 关键词 + URL | 关键词与 URL | 共同约束主题和页面上下文。 |

任务字段：任务名称、项目、Ads 账号、挖词模式、种子词/URL、国家/地区、语言、搜索网络、是否包含成人词、否定词、备注。

保存字段：关键词、近似变体、`avg_monthly_searches`、`monthly_search_volumes`、`competition`、`competition_index`、低/高页首出价（micros）、币种、请求参数、采集日期、原始响应引用。

#### 调用限制

- 使用异步队列，不在 HTTP 请求内等待结果。
- 以 Customer ID 限流，初始建议 1 QPS，可配置；以 API 实际 quota/rate-limit 错误为准。
- 相同任务配置在 30 天内提示用户查看已有结果或确认重跑。
- 网络错误、临时 5xx、可恢复的配额错误采用指数退避；授权/参数错误不无限重试。

### 4.3 Google Ads 下载文件导入（P0）

支持 CSV（UTF-8、UTF-8 BOM、GBK）和 XLSX；一期不支持 XLS。默认限制：10 MB / 10 万行，服务端可配置。

导入步骤：

1. 选择项目和导入类型（Google Ads 导出/通用关键词表）。
2. 上传文件并预览前 50 行。
3. 自动识别表头，用户可手动映射字段。
4. 指定国家、语言、币种、指标日期与来源标签。
5. 预校验有效、重复、异常行。
6. 确认入库，显示结果并允许下载错误行。

| 标准字段 | 必填 | 常见表头 |
| --- | ---: | --- |
| `keyword` | 是 | Keyword / 关键词 / Search term |
| `avg_monthly_searches` | 否 | Avg. monthly searches / 月均搜索量 |
| `competition_level` | 否 | Competition / 竞争度 |
| `competition_index` | 否 | Competition index / 竞争指数 |
| `low_top_of_page_bid` | 否 | Top of page bid (low range) |
| `high_top_of_page_bid` | 否 | Top of page bid (high range) |
| `cpc` | 否 | CPC / 平均 CPC |
| `country`、`language`、`metric_date` | 否 | 文件列或导入时指定 |
| `group`、`note` | 否 | 自定义列 |

必须保存原文件名、文件哈希、导入人、导入时间、映射版本、原始行与错误原因。相同词再次导入时新增指标快照与来源，不能直接覆盖历史。

### 4.4 Google 下拉建议词协议挖掘（P0）

#### 定位与合规边界

该来源用于发现用户常见表达和长尾词，不是面向开发者承诺稳定性的官方 API。因此必须设计为可关闭、可替换的协议适配器，并在后台标记为“非官方协议数据源”。

禁止抓取 Google 搜索结果页、自动化浏览器、使用登录 Cookie、破解验证码、轮换代理或规避访问限制。采集前应由产品负责人确认当前 Google 条款与所在地合规要求。

#### 一期协议

首选请求：

```http
GET https://www.google.com/complete/search
  ?client=firefox
  &q={UTF-8 URL 编码的查询词}
  &hl={zh-CN / en 等语言}
  &gl={CN / US 等国家}
```

备用域名：`https://suggestqueries.google.com/complete/search`。

`client=firefox` 通常返回如下 JSON：

```json
["seo 工具", ["seo 工具推荐", "seo 工具有哪些", "seo 工具免费"]]
```

解析器仅读取数组第 2 项（索引 `1`）中的字符串列表。要保存请求参数、协议版本、响应耗时和原始响应引用；若结构异常、返回 HTML 或验证码特征，应记录“协议响应变更/受限”，不影响其他任务。

适配器标准输入：`query`、`hl`、`gl`、`client_profile`、`timeout_ms`。标准输出：`success`、`suggestions[]`、`http_status`、`response_time_ms`、`error_code`、`retryable`、`raw_response_ref`。协议配置要版本化，例如 `google_suggest_v1`，后续可灰度切换。

`hl`、`gl` 只表示语言/国家倾向；建议词仍可能受出口 IP、时间和 Google 实验策略影响，界面应写作“协议采样结果”，不可承诺是当地用户的绝对真实下拉词。

#### 任务参数与扩展策略

| 参数 | 规则 |
| --- | --- |
| 种子词 | 必填；支持换行粘贴，创建前去重预览。 |
| 国家/语言 | 必填，分别映射 `gl`、`hl`。 |
| 扩展模式 | 仅种子词 / 标准扩展 / 深度扩展。 |
| 最大层级 | 默认 1，最高 2。 |
| 每种子最大请求数 | 默认 30。 |
| 语言词包 | 字母、数字、疑问词和意图修饰词，按市场分别配置。 |
| 品牌词、否定词 | 命中后标记排除而不删除原始记录。 |
| 重复策略 | 默认复用 7 天缓存；可强制重采。 |

标准扩展仅做有限请求：

- 原词：`seo 工具`；
- 字母/数字后缀：`seo 工具 a` … `seo 工具 z`、`seo 工具 0` … `seo 工具 9`；
- 语言词包：中文如“推荐、价格、哪个好、怎么、如何”，英文如“best、review、price、vs、how to”；
- 常用前缀：如“免费 {种子词}”、“best {seed}”。

深度扩展只选择第一层中“新词、高相关、未命中否定词”的少量建议词作为二级种子；每个原始种子最多选 20 个二级词，禁止超过两层。

结果字段：原始建议词、规范化词、原始种子、父查询词、扩展规则、扩展层级、建议位置、`hl`、`gl`、首次/最近发现时间、出现次数、原始响应引用、处理状态。

#### 缓存、限流与失败

- 缓存键：`provider + q + hl + gl + protocol_version`，默认缓存 7 天。
- 同一任务内相同缓存键只请求一次；缓存命中时显示原采集日期。
- 每任务默认预算 100–300 次请求；低并发队列，初始 1–2 个并发。
- 超时和 5xx 最多重试 2 次，采用指数退避。
- 收到 429、403、验证码或异常 HTML 时，立即暂停该数据源并提示管理员；不规避限制。
- 无建议词是“成功但无结果”，不是失败。

### 4.5 清洗、去重与关键词库（P0）

处理顺序：

1. 保存原始行或原始响应。
2. Unicode 规范化，移除不可见字符，清理首尾/连续空格，统一全半角。
3. 英文规范词转小写；中文保留原展示词。
4. 校验空词、异常字符、超长词（默认 2–120 字符）和数值格式。
5. 应用品牌词、否定词、敏感词规则；命中后保存并标记排除原因。
6. 依据唯一键合并关键词主记录，分别保存来源和指标快照。

关键词库列表列：关键词、国家/语言、来源、种子词、月均搜索量、月度趋势、竞争度、出价范围、机会分、标签、状态、首次发现、最近更新。

支持按关键词、国家、语言、来源、种子词、任务/导入批次、搜索量、竞争度、出价、标签、状态、更新时间筛选与排序。批量操作：加/移标签、改优先级、改状态、导出 CSV/XLSX、软删除/恢复。

状态：`待筛选`、`已保留`、`已排除`、`已归档`。所有人工修改都要写审计日志。

### 4.6 机会分（P0）

机会分仅用于词库排序，不代表排名成功概率；默认 0–100 分，必须展示分项：

| 维度 | 默认权重 | 规则 |
| --- | ---: | --- |
| 月均搜索量 | 40% | 对数归一化。 |
| 商业信号 | 25% | CPC/页首出价。 |
| 广告竞争度 | 20% | 竞争度越低得分越高。 |
| 来源丰富度 | 10% | 多来源发现适度加分。 |
| 人工优先级 | 5% | 用户手工调整。 |

缺少 Ads 指标时，仅用已知维度重新归一化。项目负责人可配置权重，配置需记录版本。

## 7. 任务、权限与安全

任务状态：`草稿 → 排队中 → 执行中 → 解析中 → 待确认入库 → 已完成`。

异常状态：`部分完成`、`授权失效`、`权限不足`、`参数错误`、`配额不足`、`受限暂停`、`文件格式异常`、`已取消`。

- 创建接口立即返回任务 ID，后台 Worker 完成取数、解析、清洗和入库；任务需幂等键。
- 管理员可连接/解除 Ads、配置协议源与限流；运营可创建任务、导入、筛选、导出；只读只可查看和导出。
- OAuth 凭据加密、日志脱敏、项目查询强制隔离；文件校验扩展名、MIME、大小和行数。
- 删除 Ads 连接立即删除凭据；是否保留历史关键词数据应在隐私政策中说明。

## 8. P0 验收标准

1. 可创建包含项目、国家、语言和种子词的挖词任务。
2. Ads 授权后能选择可访问账号，获得建议关键词及可返回的 Ads 指标。
3. 可导入 Ads CSV/XLSX，完成字段映射、校验、入库和错误行下载。
4. 可通过 Google 下拉协议采集建议词，支持原词、字母/数字、意图修饰词扩展，并展示种子、父查询、规则和层级。
5. 下拉词任务可设置 `hl`、`gl`、最大层级、请求上限；不同市场结果可独立查看与导出。
6. 同一关键词多次采集不产生重复主记录，仍完整保留多来源、快照和发现路径。
7. 词库可按词、市场、来源、种子、搜索量、竞争度和状态筛选，支持批量标记和导出。
8. 429、验证码/异常 HTML 自动暂停下拉词数据源，不尝试规避限制；任务有错误日志和安全的重试入口。
9. 页面和接口中不出现内容、发布、GSC、排名、内链等非一期功能。

## 9. 建议开工顺序

1. 项目、关键词/来源/指标快照表、任务框架与关键词库页面。
2. CSV/XLSX 导入、字段映射、清洗、去重、导出。
3. Google 下拉词协议适配器、队列、缓存、扩展策略、限流保护。
4. Google Ads OAuth、账户选择与 `GenerateKeywordIdeas` 任务。
5. 机会分、标签、状态、审计日志和验收测试。

先完成第 1–3 步，即可交付“种子词 → 下拉长尾词 → 关键词库”的可用版本；Google Ads API 资质准备完成后再接入第 4 步。
