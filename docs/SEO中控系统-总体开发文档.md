# SEO 中控系统总体开发文档

> 版本：v1.0  
> 定位：后续所有功能开发的上位规范。  
> 使用方式：先按本文确认模块归属、数据对象、页面、权限与验收边界；再为具体模块编写下级开发文档和测试。  
> 现状：关键词挖掘基础架构与网页工作台已落地；其余模块为规划范围，尚未实现。

## 1. 产品目标与闭环

本系统面向多网站、多国家/语言市场的 SEO 团队，把分散的 SEO 工作组织为可追踪闭环：

```text
关键词机会 → 内容规划/生成 → 审核与发布 → 搜索表现/排名跟踪
      ↑                                             ↓
      └────── 优化建议 ← 内链/技术检查 ← 任务执行与效果复盘 ──────┘
```

系统负责数据、分析、建议、流程和可控执行；不替代 CMS、Google Ads、开发运维平台或通用 BI。

### 1.1 全局维度

所有功能必须明确以下维度，避免数据混用：

- **组织**：最高的数据和权限隔离单位。
- **项目**：一个客户、品牌或业务线。
- **站点**：域名、子域或语言站。
- **市场**：搜索引擎、国家/地区、语言、设备、搜索类型。
- **时间**：数据快照、趋势、任务效果都必须有可追溯日期。
- **关键词、URL、内容、任务**：系统四个核心运营对象。

### 2.2 全局不做事项

- 不绕过验证码、访问限制或网页登录保护采集 Google/竞品数据。
- 不将 Google Ads 搜索量、GSC 平均排名、第三方排名混为一个口径。
- 不在无审核、无权限时自动发布、下线页面或批量修改内链。
- 不替代网站开发、服务器运维、广告投放或企业 OA。
- 不把未经来源、时间、市场标识的数据作为优化结论。

## 3. 统一开发规则

### 3.1 数据规则

1. 所有业务数据必须归属组织，项目/站点数据必须可追溯至组织。
2. API 和跨模块关联使用不可业务化 ID（推荐 UUID/ULID）；所有时间以 UTC 保存、按项目时区显示。
3. **当前状态**与**历史观测**分表：状态、负责人、当前评分可以更新；GSC、排名、爬取、关键词指标、规则评分只能追加快照，不覆盖历史。
4. 外部数据必须保存：数据源、同步批次、原始记录引用、获取时间、请求/规则版本。
5. 关键词、URL、锚文本先标准化再比较；展示时保留原值。
6. 删除默认软删除；OAuth token、API Key 仅加密存储，审计日志不得记录明文。

### 3.2 测试与交付规则

每项功能均遵循红绿测试：

1. 先在 `tests/` 写可运行的失败测试，定义输入、输出、异常和验收行为。
2. 先确认红灯，再写最小主代码。
3. 模块测试和完整测试套件均绿灯才可合并。
4. 每个异步任务需有幂等键、可重试状态、日志和错误摘要。
5. 所有数据页应有加载中、无数据、权限不足、同步延迟、同步失败五种明确状态。

## 4. 网页信息架构与数据页总表

### 4.1 全局导航

```text
概览
关键词：挖词任务 / 关键词库 / 主题分组 / 关键词机会
内容：内容规划 / 内容库（页面库）/ 内容优化 / 发布记录

排名：排名监控 / 关键词蚕食

协作：任务中心 / 报表中心
设置：项目与站点 / 数据源与授权 / 同步记录 / 成员权限
```

### 4.2 所有数据页的统一能力

- 全局范围筛选：项目、站点、国家/地区、语言、设备、搜索类型、日期范围。
- 每个指标展示：当前周期、上一周期、环比、数据来源、最后更新时间。
- 支持搜索、筛选、排序、分页、保存视图、复制链接、CSV/XLSX 导出。
- 关键记录可下钻：关键词 → URL → 内容/表现/任务；问题/机会 → 证据 → 创建任务。
- 项目时区统一展示；GSC 明确标注数据截至日期与 2–3 天延迟。

## 5. 功能模块详细边界

### M01 项目、站点、权限

- 管理组织、成员、角色、项目、域名、目标市场、时区、币种、品牌词/否定词和告警阈值。
- 管理数据源授权的归属，不承担关键词/内容/排名分析本身。
- 角色建议：组织管理员、项目管理员、SEO 策略、内容编辑、审核/发布、开发、只读。

### M02 数据源与数据中心

- 管理 GSC、Google Ads、CMS、文件导入、排名服务、爬虫等连接。
- 统一支持授权、资源绑定、同步计划、失败重试、数据质量、原始记录追溯。
- 数据中心只回答“数据从哪里来、是否完整可靠”，不直接给出优化结论。

### M03 管理驾驶舱

- 只展示已定义口径的核心指标、趋势、风险和待办；不编辑原始数据，不直接发布内容。
- 每张图/指标卡必须能下钻至对应数据页或任务。

### M04 关键词挖掘与词库

- 来源：Google Ads Keyword Planner、Google 下拉建议词协议、文件导入、人工录入；后续加入 GSC 查询词。
- 处理：标准化、去重、品牌/否定词过滤、指标快照、意图/标签/分组、关键词-URL 绑定和机会分。
- Google 下拉词仅是发现来源，不能展示为有 Ads 搜索量或排名的权威数据。
- 详细规范以 [一期-关键词挖掘模块-开发文档.md](一期-关键词挖掘模块-开发文档.md) 为准。

### M05 内容规划与 AI Skill

- 从关键词主题生成选题、Brief、内容日历和内容版本。
- Skill 包括标题、Meta Description、大纲、正文、FAQ、改写、扩写、翻译/本地化；每次生成要保存模板/模型/提示词版本。
- AI 内容默认是草稿，不能绕过审核自动标记为已发布或 SEO 合格。
- 标题生成是内容模块的首个子功能：一个关键词可以生成多个标题候选，但只允许人工选定一个当前标题；详见 [二期-标题生成模块-开发文档.md](二期-标题生成模块-开发文档.md)。

### M06 CMS 发布

- 第一优先连接 WordPress；支持草稿、定时发布、更新、撤回、失败重试、版本回滚和远端 ID/URL 回写。
- 发布前校验审核状态、必填字段、URL 冲突、链接和 Schema；无发布权限不能修改线上页面。

### M07 GSC、排名与索引

- GSC 按查询词、页面、国家、设备、日期、搜索类型分析点击、曝光、CTR、平均排名。
- 独立排名跟踪针对指定关键词/市场/设备/预期 URL 记录排名快照。
- GSC 平均排名是聚合指标，不等于实时固定 SERP 名次；两者必须分表、分页面展示。

### M08 SEO 优化建议

- 规则：排名 4–20、高曝光低 CTR、排名下跌、内容过期、缺少关键词覆盖、蚕食、内链不足、技术问题。
- 每条建议展示触发数据、时间、URL、关键词、规则版本、优先级与推荐动作。
- 建议只能“接受/忽略/转任务”，不得直接自动改线上页面。

### M09 内链与锚文本

- 从站点爬取或 CMS 内容生成链接图谱，识别孤儿页、弱内链页、失效链接。
- 推荐来源页、目标页、插入上下文与锚文本；控制精确匹配、部分匹配、品牌词和自然锚文本比例。
- 审核后才创建 CMS 更新任务，保留变更和回滚记录；不支持无差别批量插精确锚文本。

### M10 技术 SEO

- 爬取并检测 404/5xx、重定向、Canonical、robots/noindex、Sitemap、元数据重复、结构化数据、性能等。
- 问题需要严重度、影响范围、证据、首次/最后发现、修复状态和复查结果。
- 系统只创建修复任务，不直接部署代码或改服务器配置。

### M11 竞品与内容差距

- 管理竞品域名、市场和人工/合规数据导入，做关键词、主题、内容类型和排名差距分析。
- 结论转化为关键词分组、内容计划或优化任务；不绕过访问限制抓取竞品。

### M12 任务、报表、预警

- 任务可由人工、关键词机会、内容计划、发布失败、技术问题、内链建议自动创建。
- 任务包含证据、关联词/URL、负责人、截止时间、状态、评论、验收与 7/14/30 天效果复盘。
- 报表支持周报/月报与异常预警；每份报表保存快照和指标口径。

## 6. 核心数据模型

### 6.1 实体总览

| 域 | 核心实体 | 关键关系/唯一原则 |
| --- | --- | --- |
| 租户与范围 | `organizations`、`users`、`organization_members`、`projects`、`sites`、`site_markets` | 组织→项目→站点→市场；站点域名和市场组合须规范化唯一 |
| 连接与同步 | `integration_connections`、`integration_bindings`、`sync_runs`、`import_files`、`import_rows` | 每条外部数据能追溯连接、范围、批次和原始记录 |
| 关键词 | `keyword_terms`、`keyword_markets`、`keyword_observations`、`keyword_metric_snapshots`、`keyword_clusters`、`keyword_url_targets` | 词本体与市场化词对象分离；观测/指标按日期追加 |
| URL 与内容 | `urls`、`url_aliases`、`content_items`、`content_versions`、`content_keyword_targets`、`publishing_jobs` | URL 以 canonical 规范化唯一；内容版本不可覆盖 |
| GSC 与排名 | `gsc_properties`、`gsc_performance_daily`、`gsc_index_snapshots`、`rank_tracking_targets`、`rank_check_runs`、`rank_snapshots` | GSC 日事实与指定词排名快照分开保存 |
| 链接与技术 | `crawl_runs`、`crawl_url_snapshots`、`link_edges`、`link_suggestions`、`technical_issues`、`technical_issue_occurrences` | 每次 crawl 是独立事实；当前图谱/问题由最新快照派生 |
| 协作与治理 | `opportunity_snapshots`、`tasks`、`task_relations`、`task_comments`、`audit_logs`、`report_snapshots` | 任务通过关联表指向任意业务对象，审计不可变 |

### 6.2 关系图

```text
组织 → 项目 → 站点 → 市场
组织 → 外部连接 → 资源绑定 → 同步批次

市场 ← 关键词市场 ← 关键词本体
关键词市场 → 观测记录 / 指标快照 / 分组 / 目标 URL / 排名目标

站点 → URL → 内容项 → 内容版本 → 发布任务
站点 → GSC Property → GSC 日事实
站点 → 爬取批次 → 页面快照 / 链接边 / 技术问题

GSC、排名、内容、链接、技术事实 → 优化机会 → 任务 → 效果复盘
```

### 6.3 关键约束

- `organization_id` 或可经 `project_id` 追溯组织，是所有业务对象的强制边界。
- `keyword_terms` 按规范化文本和语言唯一；`keyword_markets` 按词和站点市场唯一。
- `urls` 按站点和规范化 canonical URL 唯一；变体 URL 进入 `url_aliases`。
- `keyword_metric_snapshots`、GSC、排名、爬取、机会均须含 `snapshot_date/observed_at` 与数据源/规则版本。
- `link_suggestions` 以来源 URL + 目标 URL + 锚文本 + 内容版本/哈希生成指纹，保证重复生成幂等。
- 所有发布、导入、同步、爬取、排名检查、建议生成的写入接口必须有幂等键。

## 7. 外部连接与异步任务

| 连接 | 首期用途 | 后续用途 | 注意事项 |
| --- | --- | --- | --- |
| Google Ads API | 挖词及市场指标 | 指标刷新 | OAuth、Developer Token、配额和条款；仅展示估算指标 |

## 9. 完成功能的通用验收清单

- 数据范围受组织/项目/站点权限约束，跨项目数据不可读取或修改。
- 页面支持全局市场/日期筛选，显示数据来源和最后更新时间。
- 外部任务具备运行、部分完成、失败、取消、重试、限流/授权失效状态。
- 原始数据、历史快照、当前状态可分别追溯；重跑不产生重复主记录。
- 关键动作（导入、发布、删除、批量内链、权限调整、导出）有权限、确认和审计。
- 提供无数据、无权限、数据延迟、失败状态和可操作提示。
- 先有失败测试，模块和全量测试都绿灯；异步链路有幂等与重试测试。
- 新增指标、评分或 AI 结论可以看到计算依据、数据/规则/模型版本。
