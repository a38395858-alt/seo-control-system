# 一期基础架构与数据库设计

> 前端技术栈已升级为 **React + Vite + TypeScript**。本文件中关于原生 `web/app.js` 的描述仅代表旧版实现；以《前端架构-React开发规范》为当前前端规范。

> 范围：关键词挖掘。  
> 技术：Python 3.11、SQLite、标准库 `unittest`、原生 HTML/CSS/JavaScript。  
> 已实现：可浏览网页工作台、SQLite 初始化、关键词基础规则、任务状态机和本地 Web 服务。

## 1. 当前架构

```text
浏览器
  ↓ HTTP
本地 Web 服务（src/seo_control/web_server.py）
  ↓
网页工作台（web/index.html + styles.css + app.js）

后续数据源适配器
  Google Ads / Google 下拉词协议 / CSV-XLSX 导入
  ↓
应用服务（下一步）
  ↓
领域规则（关键词标准化、过滤、任务状态机）
  ↓
SQLite 数据库（迁移、外键、唯一约束）
```

前端已是网页，不是桌面程序。当前页面提供一期工作台骨架：项目概览、数据源、挖词任务和关键词库；示例数字用于展示布局，尚未连接真实数据接口。

## 2. 项目结构

```text
src/seo_control/
  __main__.py                 # init-db / serve 命令
  domain/
    keywords.py               # 标准化、去重键、否定词与长度校验
    research_tasks.py          # 数据源验证与任务状态机
  infrastructure/
    database.py               # SQLite 迁移与连接初始化
  web_server.py               # 静态网页服务
web/
  index.html                  # 关键词挖掘工作台
  styles.css                  # 响应式视觉样式
  app.js                      # 轻量页面交互
tests/                        # 每项主代码均先由这里的测试定义
docs/                         # 产品、架构与数据库说明
```

## 3. 如何启动

初始化数据库：

```powershell
$env:PYTHONPATH = (Join-Path (Get-Location) 'src')
python -m seo_control init-db --database data/seo-control.sqlite3
```

启动网页：

```powershell
$env:PYTHONPATH = (Join-Path (Get-Location) 'src')
python -m seo_control serve --host 127.0.0.1 --port 8000
```

然后在浏览器打开：`http://127.0.0.1:8000/`。

## 4. 已实现的领域规则

### 关键词入库前处理

- Unicode NFKC 规范化，统一全半角。
- 去除不可见字符并折叠连续空白。
- 英文大小写标准化；中文原文本保留。
- 按 `项目 + 规范词 + 国家 + 语言` 去重。
- 支持最小长度、品牌词/否定词排除，并保存排除原因。

### 挖词任务

一期只允许 `google_ads`、`google_suggest`、`file_import` 三种来源。

```text
draft → queued → running → parsing → awaiting_import → completed
                    ├────────────→ failed
                    └────────────→ paused_rate_limit → queued
```

- Google Suggest 要求种子词、`hl`、`gl`。
- Google Ads 要求语言和至少一个地理目标。
- 被限流后进入 `paused_rate_limit`，不能直接绕过限制。

## 5. 数据库

`initialize_database(path)` 每次都启用 SQLite 外键，并按 `schema_migrations` 版本表幂等执行迁移。

| 表 | 用途 |
| --- | --- |
| `projects` | 项目及默认国家、语言、币种、否定词。 |
| `keyword_research_tasks` | Ads、下拉词和文件导入任务及执行统计。 |
| `keywords` | 项目关键词主库与当前状态。 |
| `keyword_sources` | 同一关键词的多来源、种子词、父查询和扩展路径。 |
| `keyword_metric_snapshots` | Google Ads/文件导入的搜索量、趋势、竞争度、出价历史。 |
| `suggest_query_jobs` | 下拉词协议请求、`hl/gl`、扩展层级、响应状态。 |
| `import_batches` | 文件哈希、列映射、成功/异常行及错误文件引用。 |
| `schema_migrations` | 已执行的迁移版本。 |

关键数据库约束：

```text
keywords:
  UNIQUE(project_id, normalized_keyword, country_code, language_code)

suggest_query_jobs:
  UNIQUE(task_id, normalized_query, hl, gl, protocol_version)

keyword_metric_snapshots:
  UNIQUE(keyword_id, source_type, metric_date, geo_set_hash, language_code)
```

## 7. 测试约定

每项功能必须遵循红绿测试：先在 `tests/` 写失败测试，确认红灯；再写最小主代码；模块测试和全量测试均绿灯才进入下一项。

```powershell
$env:PYTHONPATH = (Join-Path (Get-Location) 'src')
python -m unittest discover -s tests -v
```
