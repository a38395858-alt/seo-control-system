# Google 下拉关键词扩展：实现说明

> 状态：已实现并通过真实协议验证。  
> 页面入口：`http://127.0.0.1:8000/`

## 使用方式

1. 在网页的“输入种子关键词”中每行输入一个词。
2. 选择建议语言（`hl`）与目标国家/地区（`gl`）。
3. 点击“开始扩展关键词”。
4. 系统请求 Google 下拉建议，并将每一层发现的新词继续扩展。

## 协议与接口

- Google 下拉协议：`https://suggestqueries.google.com/complete/search?client=firefox&q=...&hl=...&gl=...`
- 本地接口：`POST /api/suggest-expansions`
- 请求字段：`seed_keywords`、`hl`、`gl`、`max_keywords`、`max_requests`、`max_depth`
- 返回字段：`keywords`、`requests_made`、`stop_reason`

## 扩展规则

- 采用广度优先递归：种子词 → 当前层建议词 → 下一层建议词。
- 以标准化关键词全局去重，重复词不会再次请求。
- 当一整层不再产生新词时自动停止。

## 安全保护

“无限扩展”在实现上等同于“自动递归直到无新词”，但必须受硬性保护：

- 默认最多 `5,000` 个关键词。
- 默认最多 `1,000` 次下拉请求。
- 默认最多 `10` 层递归。
- 达到关键词数、请求数、层级任一上限时安全停止，并返回停止原因。
- 页面请求失败时显示错误；协议响应异常不会被当作正常结果。

这些上限可由 API 调用方配置，但不应移除，避免请求无限循环或触发 Google 访问限制。
