# SEO 中控系统二期：标题生成模块开发文档

> 版本：v1.0
> 更新：2026-07-17
> 范围：从已审核的关键词生成、筛选并确定一个内容标题。
> 前置依赖：一期关键词库、关键词 AI 审核与分类。
> 不包含：文章正文、大纲、CMS 发布、GSC 数据或排名追踪。

## 1. 模块目标

把词库中适合做 SEO 内容的关键词，转成可人工确认的内容标题。标题是内容生产的起点，而不是自动发布指令。

```text
已审核通过的关键词
        ↓
生成多个标题候选
        ↓
规则检查 + AI 说明 + 人工比较
        ↓
为该关键词选定一个标题
        ↓
后续可进入大纲 / 正文生成
```

### 1.1 核心业务规则

1. **一个关键词可生成多个标题候选，但只能选定一个标题。**
2. 一个关键词已有选定标题后，再选新标题必须先确认“替换选定标题”；系统保留原候选和选择历史，不物理删除。
3. 标题候选仅属于生成它的关键词。不同关键词即使得到相同标题，也必须提示重复风险，不能静默合并。
4. 只有关键词状态为 `approved`（已通过 SEO 审核）才允许生成标题；未审核、被排除或已删除的词不可生成。
5. 人工编辑过的候选标题也必须进入同一候选列表，随后才能被选定；禁止绕开候选记录直接写入“最终标题”。
6. “选定标题”代表内容选题已确认，不代表已生成文章、已审核或已发布。

### 1.2 本期不做

- 不根据标题直接生成整篇文章或自动发布到 CMS。
- 不承诺标题会获得排名、流量或真实搜索量。
- 不一次为数千关键词调用 AI；首版按单关键词生成，后续再设计受预算限制的批量任务。
- 不用 AI 输出替代品牌、合规、事实准确性和人工编辑审核。

## 2. 用户流程

### 2.1 从关键词库生成标题

`关键词库 → 选择一条已审核关键词 → 生成标题 → 选择标题类型和数量 → 查看候选 → 选定一个标题`

默认生成 8 条，允许范围 3–20 条。每次生成均形成一个批次，用户可保留旧批次并继续生成新批次。

### 2.2 选定与替换

1. 用户在候选卡片中点击“选定此标题”。
2. 若该关键词没有已选标题，系统将该候选标记为 `selected`。
3. 若已有已选标题，弹窗同时展示旧标题和新标题；确认后在同一事务中撤销旧选择、选定新标题，并写入审计事件。
4. 页面在关键词库显示“已选标题”和选定时间；点击可进入标题详情。

### 2.3 人工补充

用户可以新建或编辑一个标题候选。保存时执行同样的长度、重复、主关键词覆盖和敏感词检查；人工标题的 `source_type` 为 `manual`，不伪装为 AI 生成。

## 3. 页面与交互设计

### 3.1 关键词库入口

关键词库新增“标题”列和动作：

| 展示状态 | 含义 | 操作 |
| --- | --- | --- |
| 未生成 | 没有标题候选 | 生成标题 |
| 待选择（n） | 已有 n 个候选，尚未选定 | 查看 / 选定 |
| 已选定 | 显示当前唯一的标题（超长省略） | 查看 / 替换 |
| 不可用 | 关键词未审核通过或已删除 | 显示原因，不可生成 |

### 3.2 标题工作台

页面顶部固定显示关键词、国家/语言、搜索意图、分类、需求预估指数和当前选定标题。页面分为三块：

1. **生成设置**：标题类型、目标读者、品牌语气、数量、附加要求和禁止词。
2. **标题候选列表**：标题正文、类型、长度、是否覆盖主关键词、质量分、AI 生成理由、来源/模型、生成时间；可选择、编辑、删除（软删除）、复制。
3. **当前选定标题**：每个关键词最多一个；可替换、取消选择或进入下一步“大纲生成”（后续模块，当前只显示占位入口）。

页面必须有加载中、无候选、AI 请求中、请求失败、AI 返回格式异常和权限不足状态。AI 失败不得丢失已保存候选。

### 3.3 标题类型

标题类型由关键词搜索意图预选，用户可修改：

| 类型 | 适用意图 | 示例用途 |
| --- | --- | --- |
| 教程指南 | informational | 怎么做、步骤、入门、清单 |
| 问题解答 | informational | 是什么、为什么、常见问题 |
| 对比评测 | commercial | 最佳、对比、评测、推荐 |
| 购买服务 | transactional | 价格、服务、方案、报价 |
| 商业调研 | commercial | 趋势、成本、选择指南 |
| 主题内容 | 未明确 | 解释型或支柱页选题 |

## 4. 标题生成与质量规则

### 4.1 输入

AI 请求的最小输入：关键词、目标语言区域（如 `en-US`、`en-GB`、`de-DE`、`fr-FR`）、国家/地区、搜索意图、关键词分类、标题类型、品牌语气、禁止词、数量和项目上下文（站点/行业，如已配置）。默认英文市场为 `en-US`，使用美国本地搜索表达与美式拼写；欧洲市场必须按具体语言区域生成，不能把美国英语直接复用于所有欧洲国家。

不向模型发送 API Key、用户密码、完整客户数据、未脱敏的外部连接配置或无关关键词库内容。

### 4.2 AI 输出契约

模型必须返回 JSON，不允许只返回 Markdown 文本。单个候选结构如下：

```json
{
  "title": "适合该关键词的标题",
  "title_type": "tutorial",
  "primary_keyword_included": true,
  "search_intent": "informational",
  "reason": "说明该标题与搜索意图和读者需求的匹配理由"
}
```

服务端负责 JSON 解析、去空白、规范化比较、去重、长度计数和质量评分；不信任模型自行给出的分数。返回格式错误时记录脱敏错误摘要和生成批次失败状态，前端显示可重试操作。

### 4.3 自动检查

每个候选保存以下检查结果，而非直接静默过滤：

- 主关键词是否完整覆盖（中文按规范化子串，英文按词边界优先）。
- 标题长度：中文建议 18–32 个汉字；英文建议 45–65 个字符。超出范围仅降分并提示，可由人工选定。
- 是否与当前关键词的其他候选重复（规范化后比较）。
- 是否与同项目已选标题高度相似；命中时提示“可能重复选题”。
- 是否命中项目禁止词、夸大承诺或不恰当的敏感表达。
- 标题类型和关键词搜索意图是否匹配。

首版质量分为 0–100，仅辅助排序：关键词覆盖 35 分、长度适配 20 分、意图匹配 25 分、唯一性 10 分、可读性规则 10 分。所有评分项和规则版本必须返回给前端；人工仍可选择低分标题，但系统要求填写或确认原因。

### 4.4 AI 同行标题研究（美国 Google）

标题页提供“AI 抓取 Google 前 20 排名标题”。系统向标题生成所分配的 AI 发送提示词：`抓取谷歌关键词 {keyword} 前20排名标题`，并要求按排名返回最多 20 条英文/美国 Google 自然结果标题、来源与结构化 JSON。结果直接显示并自动作为本次标题生成的 `competitor_titles`，无需用户打开 Google 或手工粘贴。

该能力依赖所选模型或中转服务提供实时联网/搜索能力；模型不能进行实时搜索时必须返回空结果与原因，不能编造排名。系统不直接爬取 Google 结果页、不规避访问限制。提示词要求模型只分析搜索意图、内容角度、读者对象和常见标题结构，**不得复制或轻微改写任何同行标题**。本次使用的标题快照随标题生成任务参数保存，便于追溯。

## 5. 数据设计

当前一期数据库保留 `projects`、`keywords`、`keyword_reviews`、`keyword_categories` 等表。本模块新增以下表，所有时间以 UTC 保存。

### 5.1 `title_generation_jobs`

记录一次生成请求和可追溯上下文。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `project_id`、`keyword_id` | 所属项目和关键词 |
| `status` | `queued` / `running` / `succeeded` / `failed` / `cancelled` |
| `request_json` | 标题类型、数量、语言、语气、禁止词等非敏感输入 |
| `provider`、`model`、`prompt_version` | AI 调用追溯 |
| `requested_count`、`generated_count` | 请求与有效候选数量 |
| `error_code`、`error_summary` | 脱敏错误信息 |
| `idempotency_key` | 防止重复提交 |
| `created_at`、`started_at`、`completed_at` | 生命周期时间 |

### 5.2 `keyword_title_candidates`

保存 AI 或人工创建的所有候选，不覆盖历史。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `project_id`、`keyword_id`、`generation_job_id` | 归属关系；人工候选的任务 ID 可为空 |
| `title`、`normalized_title` | 展示与去重比较文本 |
| `title_type`、`search_intent`、`reason` | 生成语义信息 |
| `source_type` | `ai` / `manual` |
| `quality_score`、`quality_details_json`、`rule_version` | 自动检查结果 |
| `status` | `candidate` / `selected` / `not_selected` / `archived` |
| `selected_at`、`selected_by` | 当前选择标记 |
| `created_at`、`updated_at`、`deleted_at` | 审计与软删除 |

强制约束：同一 `keyword_id` 在未删除候选中只能有一条 `status = 'selected'`。SQLite 通过部分唯一索引实现：

```sql
CREATE UNIQUE INDEX uq_one_selected_title_per_keyword
ON keyword_title_candidates(keyword_id)
WHERE status = 'selected' AND deleted_at IS NULL;
```

### 5.3 `keyword_title_selection_events`

记录选择、替换、取消和人工修改，字段包括：`id`、`project_id`、`keyword_id`、`previous_candidate_id`、`selected_candidate_id`、`action`、`reason`、`actor_id`、`created_at`。本表只追加，不更新历史。

### 5.4 关键索引与一致性

```text
keyword_title_candidates(project_id, keyword_id, created_at DESC)
keyword_title_candidates(project_id, normalized_title) WHERE deleted_at IS NULL
title_generation_jobs(project_id, keyword_id, status, created_at DESC)
title_generation_jobs(project_id, idempotency_key) UNIQUE
```

选择/替换操作必须在单一数据库事务中执行：验证关键词可用 → 校验候选归属 → 将旧 `selected` 改为 `not_selected` → 将新候选设为 `selected` → 写选择事件。任一步失败时整体回滚。

## 6. API 契约

| 方法与路径 | 用途 | 关键返回 |
| --- | --- | --- |
| `GET /api/keywords?project_id=...` | 关键词库列表 | 新增 `selected_title`、`title_candidate_count`、`title_status` |
| `GET /api/keywords/{id}/title-candidates` | 读取该词候选与当前选择 | 候选、检查结果、选择历史 |
| `POST /api/title-generation-jobs` | 创建生成任务 | `job_id`、`status=queued` |
| `POST /api/serp-title-research` | AI 抓取单个关键词最多 20 条 Google 标题 | `keyword`、`titles[]`、可选 `warning` |
| `GET /api/title-generation-jobs/{id}` | 轮询生成任务 | 状态、进度、候选数、脱敏错误 |
| `POST /api/title-candidates` | 手工新增候选 | 新候选及质量检查结果 |
| `PATCH /api/title-candidates/{id}` | 编辑未删除候选 | 更新后的候选及重新评分 |
| `POST /api/title-candidates/{id}/select` | 选定或替换标题 | 当前唯一选定标题、选择事件 |
| `POST /api/title-candidates/{id}/unselect` | 取消当前选择 | 无选定标题状态 |
| `DELETE /api/title-candidates/{id}` | 软删除候选 | 不允许删除当前已选标题，须先取消/替换 |

所有写接口必须验证：候选、关键词和项目三者一致；请求带 `project_id`；删除使用软删除；错误响应不暴露 AI Key、完整提示词或上游原始响应。

## 7. AI 与任务执行

- 默认复用现有 OpenAI 兼容配置：`SEO_AI_API_KEY`、`SEO_AI_BASE_URL`、`SEO_AI_MODEL`；可用于 OpenAI 或 DeepSeek 兼容接口。
- 前端不接收、保存或展示密钥。服务端日志只记录 provider、model、耗时、HTTP 状态和脱敏错误。
- 每次生成创建异步任务，前端轮询任务状态；同一关键词的相同配置短时间重复提交由幂等键合并。
- 网络超时、429、5xx 可按有限次数指数退避；鉴权、参数或模型 JSON 格式错误不自动无限重试。
- 失败后保留任务记录和之前成功的候选，允许用户发起新的生成批次。

## 8. 权限与审计

- 项目管理员、SEO 策略、内容编辑：生成、创建、编辑、选定和取消标题。
- 审核/发布角色：可查看和确认选定标题；是否可替换由项目权限配置决定。
- 只读角色：仅查看、复制和导出。
- 生成、编辑、选定、替换、取消、软删除均写审计事件；记录操作者、时间、对象、前后状态和原因。

当前本地单用户版本可先以默认操作者写入，后续接入成员权限时保持 API 与审计字段兼容。

## 9. 红绿测试与验收

遵循项目测试规则：先在 `tests/` 增加失败测试（红灯），再实现最小代码，模块测试和完整测试均绿灯后才合并。

建议新增：

```text
tests/test_title_generation_service.py
tests/test_title_generation_api.py
tests/test_title_selection_repository.py
tests/test_title_generation_web.py
```

必须覆盖以下验收场景：

1. 已审核通过关键词可创建标题生成任务；未通过关键词返回明确错误。
2. 一个任务可保存多条有效候选，AI 返回重复/空标题会被标记或拒绝，不能造成脏数据。
3. 同一关键词第一次选定标题成功；第二次选择在确认替换后，旧标题变为 `not_selected`。
4. 数据库任何时候都不能存在一个关键词两条未删除的 `selected` 候选；并发选择时只有一个事务成功。
5. 手工标题进入候选表、执行质量检查并可被选定。
6. 删除当前选定标题被拒绝；取消选择后才能软删除。
7. 标题相似/重复、关键词未覆盖、长度异常、禁止词命中均有可读提示；低分标题经确认仍可选定。
8. AI 网络失败、429、非 JSON 响应和请求超时显示脱敏错误，可重试，且不影响已有候选。
9. React 页面显示生成中、无数据、失败和当前唯一选定标题；关键词库能显示标题状态。
10. `npm run build` 与 `python -m unittest discover -s tests` 均通过。

## 10. 开发顺序

1. **数据迁移与领域规则**：建三张表、唯一约束、候选质量规则与选择事务；先完成单元测试。
2. **后端 API**：任务创建、状态查询、候选列表、人工候选、选定/替换/取消；完成 API 契约测试。
3. **AI 适配器**：复用兼容模型连接，定义 JSON 提示词与异常处理；使用模拟客户端完成测试。
4. **React 页面**：关键词库入口、标题工作台、候选卡片、选择确认与状态反馈；构建验证。
5. **真实服务调试**：在用户本地配置的模型服务上小样本测试，确认模型名、返回结构和日志脱敏，不写入任何密钥。

## 11. 完成定义

当用户可以从已审核关键词进入标题工作台，得到多个可追溯候选，并且为每个关键词只选定一个标题、可安全替换且全过程有日志时，标题生成模块即完成。文章大纲和正文生成功能应只消费这个“当前选定标题”，不自行再为同一关键词创建第二个最终标题。
